## 🛠️ 私の開発バックグラウンド（これまでの経歴）
私は「自分の手でロジックを組み立て、形にするモノ作り」が昔から大好きです。  
これまでの開発経験と、そこで得たエンジニアとしての原点をまとめました。

### 🎮 1. Minecraftでのゲームシステム・空間設計（開発の原点）
* **中学校〜高校時代（初代システムの構築）**:  

  友人たちと「自分たちが理想とするルールの人狼ゲーム」を遊ぶため、既存の配布ワールドに頼らず、  
  コマンドブロックを用いた独自のアルゴリズム構築を決意しました。  
  当初は知識不足による挫折も経験しましたが、「ロジックを形にしたい」という強い思いから、  
  情報系を専門的に学べる学校へと進学。
  「PCを所有していない友人たちも全員で遊べるように」というターゲットユーザーの環境（制約・UX）を考慮し、  
  あえて関数の縛りが厳しい**統合版（Bedrock Edition）**での開発に挑戦しました。  
  また、バグの修正（デバッグ）を何度も泥臭く繰り返し、高校3年時に完全完成させました。  
* **データ消失と『ケイドロ Ver.2』への大刷新（現在進行形での開発）**:  

  自作PCへの環境移行時のトラブルにより、初代システムとワールドデータがすべて消失するというアクシデントを経験しました。  
  しかし、この「全ロス」を機に折れることなく、現在はさらに仕様を拡張した『ケイドロシステム Ver.2』を友人と共に現在進行形で共同開発しています。
  Ver.2では、平面的なマップから脱却し、**「100×100×100h（3階建て）」の広大な立体建造物マップ**へと完全刷新。  
  中央のシンボル建物を軸に、空間を以下のアーキテクチャで階層化・システム制御しています。

  - **1階 (下層)**: 泥棒陣営の初期スポーン位置。
  - **2階 (中層)**: 警察に確保された泥棒が条件分岐によって転送される「牢獄」エリア。
  - **3階 (上層)**: 警察陣営の初期スポーン位置。
  
  現在、ゲーム開始前の「待機場所（ラウンジ）」による状態制御や、警察専用のテレポートギミックなど、より高度なイベントトリガーの実装を進めています。

### 🌐 2. 商業×IT：架空EC・コーポレートサイトの構築（高校時代の取り組み）
商品の発注、デジタル帳簿管理（簿記）、消費者とのメール対応、発送業務といった「一連のECサプライチェーン（業務フロー）」をデジタル上でシミュレーションする授業において、 
その総仕上げとして自社Webサイト（架空企業：霧ヶ峯岸株式会社）の設計・開発を担当しました。

* **仕様と特徴**:
  - **インタラクティブなUXの実装**: `onmouseover` / `onmouseout` を活用し、  
  商品一覧（エアコン、洗濯機、扇風機等）やメニューアイコンにホバーエフェクト（画像の切り替え）を実装。  
  ユーザーへの視覚的なフィードバックを強化しました。
  - **外部連携とフォーム設計**: JavaScript（`window.open`）を用いて、拠点の位置情報をGoogle Mapsと動的に連携。  
  また、お問い合わせフォームでは `type="email"` や `select` / `textarea` を適切に組み合わせ、  
  実務に耐えうるデータ入力インターフェースを構築しました。
* **得られた成長**:
  ただコードを書くのではなく、  
  **「裏側にあるビジネスの仕組み」を理解した上で、それを表現するためのシステムを構築する**という、  
  現在の大学での情報通信エンジニアリングの根底となる「目的思考の開発スタンス」を養うことができました。

## 🎯 なぜ今、Flutter（モバイルアプリ開発）なのか？
これまでゲームシステムやWebなどの開発に触れてきましたが、「スマートフォンアプリ」という、より日常に密着したUI/UXを通して、  
**身近な人の不満やリアルな課題を解決できるプロダクトを作りたい**と考えるようになったからです。

数あるフレームワークの中から、クロスプラットフォームで美しいUIを効率的に構築できる「Flutter（Dart）」を選定し、  
まずはハンズオンでの基礎習得（コンポーネントの扱い、画面遷移、状態管理など）を目標に本リポジトリを始動しました。

このハンズオンでアプリ開発の基礎体力を身につけた後、以下のオリジナルプロダクトの開発へ移行する計画を立てています。
